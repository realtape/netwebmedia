## 🔴 Critical findings

### 1. www subdomain not enforcing canonical host

Per `CLAUDE.md` and on-page `<link rel=canonical>`: canonical host is **non-www**. But the live behavior:

```
https://www.netwebmedia.com/                          → 200 OK (full HTML served)
https://www.netwebmedia.com/services.html             → 200 OK
https://www.netwebmedia.com/pricing.html              → 200 OK
https://www.netwebmedia.com/blog/aeo-replaces-seo-2026.html → 200 OK
```

Every URL is fully duplicated under `www`. Google sees two parallel sites. PageRank is split. Backlinks pointing at `www.` don't accrue to apex.

**Fix:** In `.htaccess`, add at the top of the rewrite block:

```apache
RewriteEngine On
RewriteCond %{HTTP_HOST} ^www\.netwebmedia\.com$ [NC]
RewriteRule ^(.*)$ https://netwebmedia.com/$1 [R=301,L]
```

Verify cPanel doesn't have the www subdomain pointed at the same docroot in a way that bypasses .htaccess. If it does, remove the www subdomain entirely or repoint it to a redirect rule.

### 2. Homepage links to forbidden internal city-prospect pages

The homepage (`index.html`) contains a row of pill-style city buttons:

```html
<a href="/la-serena-prospects-report.html">La Serena</a>
<a href="/antofagasta-prospects-report.html">Antofagasta</a>
<a href="/concepcion-prospects-report.html">Concepción</a>
... (likely 14 more)
```

These pages are:
- **403'd** by `.htaccess` (`*-prospects-report.html` pattern)
- **Disallowed** in `robots.txt`
- **Internal-only** per CLAUDE.md (named-prospect lists, sales asset)

Result: any visitor or crawler clicking these pills gets `403 Forbidden`. From a UX standpoint that's a broken page on the homepage. From an SEO standpoint Google wastes crawl budget hitting forbidden URLs from a high-priority page.

**Fix:** Either remove the city-pill block from `index.html`, or replace each link with a public city landing page (e.g. `/industries/restaurants?city=concepcion`). I lean remove — the campaign these pills came from has shifted, and the public site shouldn't expose Chile-only sales artifacts on its US-targeted homepage.

### 3. Meta Pixel placeholder, never initialized

```javascript

fbq('track', 'PageView');
```

`window.NWM_META_PIXEL_ID` is never set anywhere in the served HTML. The pixel script loads, the init call fires with `undefined`, the track call fires against a non-pixel — and Meta drops all of it. **Zero conversions tracked from Facebook/Instagram traffic right now.**

### 4. Sitemap ↔ canonical mismatch

`sitemap.xml` lists 190 URLs, mostly extensionless:

```xml
<loc>https://netwebmedia.com/about</loc>
<loc>https://netwebmedia.com/aeo-agency</loc>
<loc>https://netwebmedia.com/blog</loc>
```

But every one of those 301-redirects to `.html`:

```
/services  → 301 → /services.html
/pricing   → 301 → /pricing.html
/about     → 301 → /about.html
/contact   → 301 → /contact.html
/blog      → 301 → /blog.html
/industries → 301 → /industries/index.html
```

And every target page declares its `.html` URL as canonical:

```html
<link rel="canonical" href="https://netwebmedia.com/services.html" />
<link rel="canonical" href="https://netwebmedia.com/pricing.html" />
```

**This contradicts CLAUDE.md** which says "Top-level pages have NO extension and NO trailing slash in canonical URLs." Production does the opposite — it canonicalizes to `.html`.

You have two clean paths:

**Option A (faster — match production to docs):** Flip `.htaccess` and canonical tags to make extensionless the canonical:
- 301 `*.html` → extensionless (currently the reverse)
- Update `<link rel=canonical>` on every top-level page to omit `.html`

**Option B (faster still — match docs to production):** Regenerate `sitemap.xml` to list `.html` URLs. Update CLAUDE.md to reflect actual behavior. Done in 5 minutes via `regen-sitemap.py`.

I'd pick **B**. The .html is already what Google has been indexing; ripping that up risks short-term ranking loss.

## 🟠 High findings

### 5. CSS/JS Cache-Control too aggressive

```
css/styles.css → Cache-Control: public, max-age=300, must-revalidate
js/main.js     → Cache-Control: public, max-age=300, must-revalidate
```

5-minute cache on assets that are already versioned with query strings (`styles.css?v=6`, `main.js?v=7`). Repeat visitors and crawlers refetch every 5 min for no benefit. Image assets are correctly set to `max-age=31536000, immutable`.

**Fix:** In `.htaccess` for CSS/JS:

```apache
<FilesMatch "\.(css|js)$">
  Header set Cache-Control "public, max-age=31536000, immutable"
</FilesMatch>
```

Bump the `?v=` query-string when you ship a CSS/JS change. (You already do: `?v=6`, `?v=7`.)

### 6. HTML Cache-Control: no-store

```
text/html → Cache-Control: no-store, no-cache, must-revalidate, max-age=0
```

This kills any chance of edge/CDN caching. For a marketing site with no per-user content, this is wasteful. Move to:

```
Cache-Control: public, max-age=0, s-maxage=300, stale-while-revalidate=86400
```

`max-age=0` keeps the browser from caching stale HTML; `s-maxage=300` lets a CDN serve cached HTML for 5 min; `stale-while-revalidate` lets stale-but-revalidating responses go out instantly during refresh.

### 7. Industry pages missing `<link rel=canonical>`

```
GET /industries/legal-services.html       → 404
GET /industries/legal-services/index.html → 200
GET /industries/legal-services            → 301 → /industries/legal-services/
```

Three URLs resolve to one page; no canonical declared in HTML. Google will pick its own and may dilute. Add to every `industries/<niche>/index.html`:

```html
<link rel="canonical" href="https://netwebmedia.com/industries/<niche>/" />
```

Edit `build_industry_pages.py` template, regenerate, push.

### 8. Duplicate Google Fonts `<link>` on homepage

```html
<link rel="preload" as="style" href="https://fonts.googleapis.com/css2?family=Inter…">
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter…">
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter…">  ← dup
```

Browser may dedupe but it's unnecessary. Remove one.

### 9. CLAUDE.md routing rule contradicts production

CLAUDE.md says canonicals are extensionless; production canonicals are `.html`. Whichever direction you pick (Option A or B above), update CLAUDE.md to match.

### 12. Mixed-case absolute/relative JS paths

Inconsistent in homepage:
```
src="/js/nwm-sentry.js"       (absolute)
src="js/main.js?v=7"          (relative)
src="js/cart.js"              (relative)
```

Works because all are at the right level for `index.html`, but breaks if HTML is served from a different path or templated. Standardize on absolute (`/js/…`).
