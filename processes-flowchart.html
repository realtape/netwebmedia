Two databases, never crossed

webmed6_nwm (api-php: leads, EAV resources, drip) and webmed6_crm (crm-vanilla: contacts, deals, workflows, conversations). Two separate PDO connections — split into two API calls when both are needed.

Cron drives everything async

Drip queue (/api/cron) and workflow engine (cron_workflows) both fire */5 min via cron-workflows.yml. cPanel cron is the fallback if GH Actions is down >1h.

AI is Claude only

Triage uses claude-haiku-4-5 via the Anthropic API with a strict enum-normalized output and a non-fatal fallback shape. ChatGPT / Perplexity / AIO are AEO targets, not internal tools.

Deploy = InMotion only

GitHub Actions FTPS → cPanel. Migrations auto-run idempotently after each deploy. Never Vercel / Netlify / other hosts.
