## User flow (end-to-end demo)

1. **Carlos logs in** → lands on `/crm-vanilla/admin.html`. `admin.js` builds the standard header; the new mount script sees `.header-right` exists, drops in the org-switcher slot, and `org-switcher.js` calls `GET /api/?r=organizations`. Because Carlos owns the master org, the API returns every org (with `member_count`); `detectMasterOwner()` flips true. The dropdown trigger appears next to the avatar; a "Sub-accounts" item is appended to the sidebar nav.
2. **Carlos opens the switcher → "Manage sub-accounts →"** (or the new sidebar link). On `subaccounts.html` he sees the table (slug · name · plan · status · members · created · actions). He clicks **+ New sub-account**, the modal auto-slugifies the name and auto-suggests `<slug>.netwebmedia.com`. POST to `/api/?r=organizations` returns the new row; toast confirms; cache busted; table reloads.
3. **Carlos clicks "View"** on the new row → `POST /api/?r=organizations&sub=switch` flips the session org, then navigates to `/crm-vanilla/index.html?org=<slug>`. With `branding.js` loaded site-wide, the dashboard re-paints with the sub-account's primary/secondary colors and logo — that's the white-label moment. Switching back uses the same dropdown.

## Cross-agent dependency

The visible payoff requires the **branding agent's** `js/branding.js` (already referenced in `admin.html`) to read the active org's `branding_logo_url` / `branding_primary_color` / `branding_secondary_color` from `GET /api/?r=organizations&slug=<active>` and write them into `--accent` / `.brand-icon` / favicon. Until that ships, switching context will work logically but the page will still look like NetWebMedia. The org-switcher already exposes the active branding hex as a swatch in the dropdown, which is enough to demo the data path without `branding.js`.

## Risks / known gaps

- **Master-owner detection is client-side and heuristic** (`member_count` field presence). The handler is the source of truth — 403 from `POST /api/?r=organizations` is the real gate. UI gating just keeps the surface tidy.
- **Plan field is create-only.** PATCH allowlist on the server doesn't include `plan`; the modal correctly disables it on edit.
- **No CSRF token in headers** — relies on the router's Origin/Referer check (`api/index.php` lines 100–107). Acceptable given same-origin SPA, but flag for hardening pass.
