02The Five Engineering Phases

Each phase has a verification gate that must pass before the next starts. If any phase slips >2 days, scope is cut from the next phase, not the current one. Sequence and gates locked in ARCHITECTURE.md §12.

W2
Phase 1 — FoundationMulti-tenant schema extensions, provisioning endpoint, host-to-org resolver, isolation test in CI. Gate: test tenant provisioned + cross-tenant test green.
W3
Phase 2 — OS Shell + BrandingVanilla-JS shell at /os/, widget grid, branding.css route, logo upload, login-by-host. Gate: two tenants render two brands; Lighthouse ≥ 90.
W4
Phase 3 — Connector layerGmail + Calendar OAuth, encrypted token store (libsodium AEAD), Slack stretch, HubSpot read-only. Gate: end-to-end email read + Calendar event create.
W5
Phase 4 — Agent orchestrationAnthropic API dispatcher, command bar, run_agent workflow step, per-org token budget. Gate: 3 agent invocations + cost tracking + budget enforcement.
W6
Phase 5 — Billing + Onboarding polishStripe Checkout, webhook state machine, 4-step onboarding wizard, design partner onboards live. Gate: Stripe test sub → webhook → active flip works end-to-end.
