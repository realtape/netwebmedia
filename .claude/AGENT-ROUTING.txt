YOUR TOKEN STATUS: ~54% remaining (must last the full week)

───────────────────────────────────────────────────────────────────
WHEN TO USE EACH AGENT (PICK ONE, NOT MULTIPLE)
───────────────────────────────────────────────────────────────────

STRATEGIC / COMPLEX DECISIONS → Use Sonnet agents:
  ✓ CMO — campaign strategy, positioning, competitive analysis
  ✓ Engineering Lead — architecture, security, tech decisions
  ✓ Carlos's assistant — executive decisions, delegation, strategy

ROUTINE WORK → Use Haiku agents (costs 1/3 the tokens):
  ✓ Finance Controller — monthly reports, cash forecast, billing
  ✓ Operations Manager — SOPs, process docs, vendor reviews
  ✓ Customer Success — renewals, QBRs, health checks
  ✓ Sales Director — proposals, pipelines, forecasts
  ✓ Project Manager — status reports, kickoff docs, timelines
  ✓ Data Analyst — dashboards, weekly reports, analysis
  ✓ Content Strategist — blog briefs, social calendars, edits
  ✓ Creative Director — design feedback, brand enforcement

───────────────────────────────────────────────────────────────────
BATCHING RULE (SAVE 30-40% ON TOKENS)
───────────────────────────────────────────────────────────────────

INSTEAD OF:
  1. "Sales Director, give me the pipeline report"
  2. [wait for response]
  3. "Sales Director, now give me the forecast"
  4. [wait]
  5. "Sales Director, now give me the activity by rep"

DO THIS:
  "Sales Director: (1) pipeline report, (2) forecast, (3) activity by rep"
  → ONE call, three answers, half the tokens

───────────────────────────────────────────────────────────────────
WEEKLY PACING (TO NOT BURN OUT)
───────────────────────────────────────────────────────────────────

Monday:   Strategic calls → CMO/Engineering/Carlos's assistant
Tuesday:  Sales pipeline → Sales Director (1 batch: all metrics)
Wednesday: Operations → Operations Manager (SOPs, processes)
Thursday:  Finance → Finance Controller (reports, cash forecast)
Friday:   Analytics → Data Analyst (weekly summary)

This spreads expensive work across the week. Don't run 3 Sonnet
agents on the same day.

───────────────────────────────────────────────────────────────────
RED FLAGS — STOP AND DELEGATE
───────────────────────────────────────────────────────────────────

❌ "Can you write an email to clients?" (unnecessary agent use)
   → You write it, or use Carlos's assistant only

❌ Running 2+ Sonnet agents in parallel
   → Queue them; ask one, wait for answer, then ask the next

❌ Asking Carlos's assistant for work a role-specific agent should do
   → "Sales Director, give me the forecast" (not "Carlos, have Sales...")

❌ Same agent, back-and-forth more than 2 exchanges
   → Batch the follow-up into ONE message

═══════════════════════════════════════════════════════════════════
BOTTOM LINE: Route to the cheapest agent that works.
If Haiku can do it, use Haiku. Save Sonnet for decisions only.
