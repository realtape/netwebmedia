# Save all documents to D: — keep C: for programs only

**Stated:** 2026-05-18 by Carlos.

**Rule:** From now on, any document/deliverable/output file I create (reports, exports,
plans, generated docs, PDFs, spreadsheets, decks, notes, scratch deliverables) goes on
the **D: drive**, never C:. C: is reserved for programs/applications only.

**Drive facts:** D:\ exists (~694 GB free as of 2026-05-18). C:\ is the system/programs drive.

**Default documents root:** `D:\Documents\` — organize into subfolders by project/topic
(e.g. `D:\Documents\NetWebMedia\`). Never dump loose files at a drive root, and never
save to the Desktop (see desktop-organization memory — still applies).

**Scope / exceptions:**
- This is about *documents I produce*, not application/config data.
- The NetWebMedia code repo stays at `C:\Users\Usuario\Desktop\NetWebMedia\` — that's a
  program/source tree, not a "document." Don't relocate the repo. Source edits commit there.
- Claude memory/config under `C:\Users\Usuario\.claude\` stays put (that's config, not a doc).
- When in doubt whether something is a "document," default to D:.
